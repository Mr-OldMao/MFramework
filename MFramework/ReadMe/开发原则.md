开发原则

1.资源与代码分离解耦，不手动挂载代码到预制体上，同时尽量一个开发者操作一个预制体，避免资源冲突

原因：使资源与代码解耦，对于版本管理，资源冲突比代码冲突更难解决，所以要尽量避免资源冲突。

2.场景中不放任何内容，避免提交冲突，所有内容都用代码进行生成，整个游戏只需一个场景作为容器即可，其他场景只作为资源编辑界面作用。

3.避免使用Resources方式加载资源，使用AssetsBundle处理资源加载
